Coding Style Conventions

  • Variable names, property names and method names are written with camel-case with the leading character lower case. 
    	onHandVC= thisClinic.supplierWarehouse.getOnHandVC()
  • Class names are written with camel-case with the leading character uppercase. 
    	class PeopleType(abstractbaseclasses.ManagedType):
	    pass
  • Module names are all lower case, with underscores where necessary for clarity. This means that source file names are also lowercase with occasional underscores. 
    	import util
	import abstractbaseclasses
	import model_generic
  • Constants intended to be accessed from other modules are written in all uppercase with underscores as necessary. 
    	GLOBAL_FOO= "foo"

	class MyClass:
	    SOME_TAG= 1
            OTHER_TAG= 2

	if someVar==module_name.MyClass.OTHER_TAG:
	    foo= module_name.GLOBAL_FOO
        else:
	    foo= None
  • Method names, function names and class-level or module-level variable names should be prefixed with underscore when they are not intended to be accessed from outside the module. 
    	def _doubleThis(val):
	    return val+val

	class MyClass:
	    def __init__(self):
	        _someVal= 1
	    def getVal(self):
	        return self._someVal
	    def getTwiceVal(self):
	        return _doubleThis(self._someVal)
