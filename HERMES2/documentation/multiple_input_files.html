Multiple Demand and Calendar Files (SVN r1094)

As of SVN repository version 1094, there exists the capability for the user to use either multiple demand files or multiple calendar files in their simulation.

There are several reasons in which this type of feature would become necessary:

  • Specifying the specific vaccine schedules required by various people types allows for a clinic to order a different amount of vaccines than what is actually consumed by the actual people type(s).
  • Specifying varying calendar types - for various people types with the same characteristics - allows for the location to predict the demand for their vaccine consumption independently of the actual consumption.

Under default conditions, either calendarfile or demandfile (denoted in the input.kvp file) are utilized both in shipping/ordering policies (which are essentially the same thing in HERMES) AND in the actual use/consumption of product.

How to Specify the Multiple Demand and/or Calendar Files

In order to use this functionality, multiple people types must first be designated in the UnifiedPeopleTypeInfo file. The people types must have the same characteristics (i.e., same randomkeynumber) as similar existing people types.

Then, either two demand files or two calendar files must be created. These files must, for the most part, look like each other, but use the different people types defined previously. For example's sake, for a scenario using two demand files, the new file is called ConsumptionDemand.csv and the original file is called Demand.csv. (For what it's worth, the previous two steps can be done in any order.)

Then, modify the relevant input.kvp file so that you specify the two data input files -- one for the shipping (what is predicted by the model) and one for the actual consumption (what the population consumes) -- in the input.kvp file, instead of a singular file used for both shipping and consumption.

Therefore, for our example above (the multiple demand file scenario), instead of, 

        demandfile = "Demand.csv"
the input file should read: 
        shippingdemandfile = "Demand.csv"
        consumptiondemandfile = "ConsumptionDemand.csv"

Conversely, for a scenario which uses multiple calendar files, instead of 

        calendarfile = "Calendar.csv"
the input file should read: 
        shippingcalendarfile = "Calendar.csv"
        consumptioncalendarfile = "ConsumptionCalendar.csv"
