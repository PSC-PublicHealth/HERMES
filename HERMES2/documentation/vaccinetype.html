Assumptions for UnifiedVaccineTypeInfo File

As of 5 March 2012, we're splitting up the Unified input files so that entries would fall in one of the following two categories:

  • Unified Vaccine Type Info - this file contains the manufacturer-specific data for vaccines commonly used in EPI regiments. Data put into this file will be denoted as the baseline EPI vaccine regiment. This file will be placed in the ./master_data/Unified folder of the SVN.
  • Country-Specific Vaccine Type Info - this file has been created for each country (and is denoted as such in the file name of the input file). These files contain country-specific data about vaccines, and will be used to override the unified vaccinetypeinfo document. This file will be placed in the ./master_data/[COUNTRY NAME] folder of the SVN.

In either case, the vaccine file contains the columns, 'Batch', 'Name', 'Abbreviation', 'Vaccine presentation', 'Method of administration', 'Doses per vial', 'Packed vol/dose(cc) of vaccine', 'Packed vol/dose(cc) of diluent', 'Vaccine price/vial', 'Vaccine price/dose', 'Price Units', 'Doses/person', 'LifetimeFreezerDays', 'LifetimeCoolerDays', 'LifetimeRoomTempDays', 'LifetimeOpenDays', 'RandomKey' (which denotes what vaccine type it actually is), 'Wastage Rate' and 'Notes'.

We are currently using the following pattern for the 'Name': 

   Tuberculosis_ID_20dose_SII
where 'Tuberculosis' is the name of the vaccine, 'ID' is the vaccination method (in this case, "intradermal"), '20dose' indicates that the vial contains 20 doses of vaccine, and 'SII' indicates the possible origin of the vaccine (Serum Institute of India), as defined by the WHO Prequalified Vaccine Database.

Unknown vaccination/administration methods can also be abbreviated as 'U'. Using this syntax, an example of the name of the vaccine with unknown administration method and unknown manufacterer could be written as: 

   RotaV3_U_1dose_U
The above example indicates that the Rotavirus version 3 (V3) vaccine, with a single-dose presentation, has an unknown administration method and an unknown manufacturer.


General Notes:

  • Categories within the column 'Batch' are either Baseline [EPI Vaccines] (as of 2011) or vaccines to be introduced (as of 2011)
  • Vaccines that are administered either intradermally or intramuscularly, which are lyophilized (freeze-dried), are given a vial of diluent (a solvent used to dissolve the vaccine powder).
    Both of these features are indicated as taking up storage in the supply chain.
  • 'Packed volume' indicates that there may exist primary/secondary/tertiary packaging for multiple vials in a package.
  • 'Vaccine Prices', unless otherwise stated by a manufacturer, was calculated using the WHO's Vaccine Volume Calculator.
  • 'Price units', unless stated otherwise, are in US dollars (USD).

Country-Specific Notes:

  • Niger's model uses vaccine types defined by assumptions for the most common "type" of vaccine used in the UNICEF EPI supply chain, as defined by the Prequalified Vaccines Database.
  • Thailand vaccine data was compiled from the powerpoint, "Modeling the Vaccine Supply Chain in Trang Province" (Thailand 2.22.10 TA.ppt)
  • Vietnam vaccine data was compiled from the spreadsheet, Copy of VPAT_Vietnam_28Jul10.xls.
  • Chad and Senegal had all of its vaccine types explicitly defined by manufacturer in the 'Notes' column of the Unified Vaccine Type sheet.
  • Kenya contained some information about the manufacturers of its vaccines, but not extensive notes, within the 'Notes' column, as defined above.

Additional Rules For Names:

  • Do not include the '*' or '+' characters. '_' and '-' are okay to use. Spaces can be used, although they are not recommended.
  • Names should contain proper punctuation & spelling.
  • If an abbreviation for the vaccine type exists that is commonly used, that can be used to define the name for the vaccine.
