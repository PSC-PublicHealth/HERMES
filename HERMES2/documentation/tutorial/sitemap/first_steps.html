Before You Start: Obtaining a PSC Account, SVN, DAV, and RedMine

1. What Exactly is the PSC? Why do we use it?

Pittsburgh Supercomputing Center, is a joint effort of Carnegie Mellon University and the University of Pittsburgh together with Westinghouse Electric Company.

The technical team out of our logistics team are staff members of the PSC. Additionally, the Python scripts needed to create HERMES are created at the PSC. A lot of our supercomputing infrastructure - including the SVN and the DAV - are hosted on PSC servers.

The PSC operates on a secure system created at CMU called the AFS (Andrew File System), which is where our files are located. To access the SVN and the DAV, we have to obtain an account that runs on the PSC's Kerberos authentication system. Instructions on how to obtain a Kerberos account are below.


2. Obtaining a Kerberos Account + Password from the PSC

3. Basics Behind the SVN, the DAV and RedMine

The best way to describe the difference between the SVN and the DAV is that the SVN tends to act as a living, breathing document -- what is usually used as a "working copy" is the most recent revision of the document. However, previous versions of the document can be found, and used, where necessary. For the DAV, which is less of a flexible structure, it acts like a 'file-share system', where the most recent version of a document is uploaded for access.

SVN

The SVN, commonly known as subversioning, is a process through which revisions of documents are kept track of in a repository, to track the progress of document/program development over time. It's most analogous to when you create multiple versions of a particular document (e.g. an Excel spreadsheet), where the most recent version of the document is the one that you're currently working on. Using an SVN allows us to actually keep track of which PSC user made which change at which time. For the purposes of our work, we use a client called TortoiseSVN. Among its features, TortoiseSVN gives us the ability to visually inspect different sets of files to assess changes made to them.

To access the SVN, you can follow the instructions here.

DAV

The DAV, known as Web Distributed Authoring and Versioning or WebDAV for short, is a method which "makes the Web a readable and writable medium"1. It allows us to upload items - documents, pictures, files - securely on the internet with a timestamp and allows other people to download what's been created, again, in a highly secure manner. For this reason, we use our Kerberos accounts to log into the DAV.

To access the DAV, you can click or highlight this link: https://dav.psc.edu/Hermes/, and enter in your Kerboros username and password.

RedMine

RedMine is defined as a flexible project management web application, written using the Ruby on Rails framework. An open source application released under the GNU general public license, it allows for programs such as HERMES to have a central database to keep track of edits, bug and development issues. It also gives users the option to share files, through both 'wiki' (encyclopedia) and "files" functionalities. RedMine gives users the ability to keep track of multiple projects (through a Gantt chart), allows for email notifications of updates and reporitng capabilities to be in a central database.

The HERMES Redmine website can be accessed here. More information about RedMine in general can be found here.

4. Linking the DAV on your Desktop

If you'd like the ability to move folders around, create new folders and/or rename files in a drag/drop interface in Windows XP or in OS X (formerly Mac's OS X Mountain Lion):

Windows XP
  1. Open "My Network Places" (either on the Desktop or in the Control Panel).
  2. Under "Network Tasks" on the left panel, click "Add a Network Place".
  3. When the wizard pops up, click "Next".
  4. Double click "Choose another network location".
  5. Type https://dav.psc.edu/Hermes/ in the address box and click"Next".
  6. If you see a certificate alert message, click "Yes" (you likely won't see this message).
  7. Enter your PSC Kerberos username and password. (Note: you'll probably need to do this twice, as it looks like that Windows attempts an unacceptable form of authentication on the first request.)
  8. Optionally rename the network place and click "Next".
  9. Click "Next" to finish.

From here, you'll be able to find the Hermes DAV in "My Network Places", both on the Desktop and in the "Save" dialogue windows. If you'd like, you can right drag the Hermes DAV icon and create a shortcut in other places so that it is more accessible.


OS-X
  1. In the Finder, select Connect to Server under the Go menu.
  2. Enter https://dav.psc.edu/facilities/ in the address box.
  3. Enter your Kerberos username and password.
