Quick-Start Document – Clarification (FOR YOUR BENEFIT)

Here's a quick definition guide to what's in the second column of the Input Keywords Generic file (the one with the heading "Type")

TYPEDefinition & How to Use the Type
int Integer - include numbers with NO decimal points (e.g. "28", "30").
float Numbers, INCLUDING decimal points (e.g. "28.0","2.3", "0.7"). Please -- observe the default number and the description when inputting float numbers!
long For our purposes, as we're not theoretical computer scientists, "long" numbers are a different type of number - where there are MANY characters, so the integers are "long" (e.g. "123456", "3478679").
TF True/False - it'll either default to "True" or "False". Follow instructions in the documentation to see the default.
string_list Include (as necessary) multiple alphanumeric text strings. Stringlist must be in QUOTES and comma-separated.
stringOrNone Either include a text string (e.g. alphanumeric characters, usually a file name for our purposes) OR leave blank ("None").
floatOrNone Either include a floating number (see "float" above) OR leave blank ("None").
longOrNone For our purposes, this is using either a different kind of number (the "long" number) - one where the integers are "long" (e.g. "123456", "3478679") - or leave blank ("None").
