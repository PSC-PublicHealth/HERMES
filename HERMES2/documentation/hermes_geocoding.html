  • Cold Chain Equipment Manager (CCEM)
  • Others
    • Niger - 2007 CCI (see below)

In the case of Niger...

We got a Cold Chain Inventory (CCI) from the WHO in 2007.

In it, we had the following levels, with geocodes for the following levels

  • Central
  • Region
  • Districts
  • Nothing for IHCs

The next step was to cross-reference between our CCI and Google Maps, but we didn't have the exact street address, but only the towns in which they were located.

We also did some general checking using the locations that we were given for the higher levels.

Decision Hierarchy (for Niger)

  1. WHO Coordinates (where there's nothing for clinics)
  2. GeoNames (where we started with clinics)

    From there:

    If found -- then we gave Lat & Long coordinates for the particular location.
    If not found -- then we didn't include those coordinates.

We made sure that we were using a populated area.

The problem is that the names that were given in the inventory was that the names were either a mixture of Anglophone, Francophone and other mismatches of names.
We also looked at alternative names, combined with fuzzy searching, to figure out if the locations that we had "matched".
When we looked up distances for Niger, we definitely used straight-line distances.
Judgment calls did have to be made with regards to distances on some locations.
Spot-checking was also done for locations with GoogleMaps.

This may never be an exact science - so it's better to get the locations which we're absolutely sure about RIGHT. For the distances which are extremely far, we spot-checked and reconsidered them, but we would nonetheless eventually use averages.

GoogleEarth, sometimes, was absymally wrong with regards to road distances in Niger.

Other Notes

  • Niamey had some administrative boundaries; we assumed that distances within Niamey were 0 kilometers away from each other.

Based on what we see above, HERE is the protocol that we're using for geocoding:

Decision-Making Protocol for Geocoding
  1. Evaluate Current Data
    • Create summary of available data
      (Write where you got it, how you got the data, how did they derive the data, etc.)
  2. Verify Current Data Set
    • Do a Comprehensive Search to See if Other Coordinate Sources Exist
    • Write a script in Google Earth (which creates a KML file which can plot it onto the map).
    • Report Our Findings - start at the top and move down.
    • Note: Both HERMES and Google Earth use coordinates in decimal format.
  3. Start looking for other places.
    • Google Maps
    • GeoNames
    • Bing Maps (as a last resort)
  4. Things to Know
    • Go for Latitude & Longitude coordinates.
    • Look at Great Circle Distances between Upper Levels
      • If Abnormal:
      • Cross-Reference
      • If Unsure:
        • Eliminate
        • Unknown Distances = Average of Other Locations in Levels
  5. Find RoadMap Distances
    1. Google Maps
    2. Bing Maps (as a last resort)
    3. ArcGIS (useful, but may be hard to learn)

      Additional Notes:

    • Run Through and Find Distances
    • Find a ratio between distances if necessary

Remember:

  • Cross-checking is always a good idea.
  • Document your workings in the NOTES column in the Stores spreadsheet.
